# Code of Conduct

## Expected Behavior

- Be respectful and constructive.
- Focus criticism on code, design, documentation, and reproducible behavior.
- Assume good faith, but be precise when discussing risk, security, or correctness.
- Keep discussions relevant to `gdb-mcp` and Linux GDB debugging workflows.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained personal attacks.
- Publishing private information without permission.
- Posting exploit details for unresolved vulnerabilities in public issues.
- Spam, trolling, or intentionally disruptive behavior.

## Security Reports

Do not report vulnerabilities in public issues. Use GitHub Private Vulnerability
Reporting:

https://github.com/BeaCox/gdb-mcp/security/advisories/new

## Enforcement

Maintainers may remove comments, close issues, block users, or take other
reasonable action to keep the project focused and safe. Enforcement should be
proportional to the behavior and the risk to users.
